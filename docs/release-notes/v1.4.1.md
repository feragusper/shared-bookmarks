# v1.4.1

## Bug fixes

### 🐛 Folder deletes from synced user now propagate correctly
- **Fixed**: when user B deleted a folder that was synced from user A,
  the delete never reached user A. The `onRemoved` event was silently
  swallowed by stale echo-suppression entries left over from the pull
  that originally created the folder.
- **Root cause**: echo entries were never consumed by `onCreated`
  (suppressed by the reconcile lock), so `onRemoved` found them and
  bailed out. Removed the `isEchoNode` check from `onRemoved` entirely
  — the lock alone is sufficient to prevent echo loops on pull-initiated
  deletes.

### ⚡ Folder delete is now a single op (was N ops)
- Deleting a folder with 40 bookmarks previously generated **81 Firestore
  writes** (one `DEL_BOOKMARK` per bookmark × duplicates + subfolder ops).
  Now it emits a **single `DEL_FOLDER` op** — the receiver uses
  `chrome.bookmarks.removeTree()` which cascades automatically.
- Tree-state cleanup in Firestore now batch-deletes all bookmarks and
  subfolders under the path prefix in one pass.
- Drastically reduces quota usage, CPU/battery drain, and sync latency.

### 🔧 Firestore rules deployed
- Added `firebase.json` and `.firebaserc` for `firebase deploy` support.
- Deployed security rules for the new `ops` subcollection.
