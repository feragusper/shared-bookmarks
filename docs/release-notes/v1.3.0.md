# v1.3.0

## What's new

### 🔄 Sync conflict fix: local deletes are now authoritative
- **Fixed**: deleting a folder or bookmark locally and syncing would
  recreate it from the remote copy — especially after quota errors left
  `lastSyncedKeys` incomplete.
- **Solution**: a new `pendingLocalDeletes` mechanism records deletions
  **immediately** in `chrome.storage.local` before any Firestore call.
  Even if the push fails (quota, network), the next reconcile knows the
  user intentionally deleted the item and will never recreate it locally.
- Folder deletes cascade to child paths automatically.
- 5 new unit tests covering the new behavior.
