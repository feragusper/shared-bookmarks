# v1.4.0

## What's new

### 🚀 Oplog-based sync engine
- **Rewritten** `sync-core.js` to use an operation-log (oplog) model
  instead of full-state diffing. Local Chrome bookmark events are now
  converted into discrete ops (`ADD_BOOKMARK`, `DEL_BOOKMARK`,
  `ADD_FOLDER`, `DEL_FOLDER`) that are pushed to Firestore and applied
  on the partner's side.
- The service worker is now a thinner I/O shell — all sync logic lives
  in the pure, testable `sync-core.js` module.
- Folder deletes cascade correctly, emitting child bookmark and subfolder
  delete ops automatically.

### 🔥 Firestore ops collection & security rules
- New `rooms/{roomId}/ops` subcollection for the oplog.
- REST helpers in `firebase/firestore.js` for writing and reading ops.
- Updated `firestore.rules` with security rules for the ops collection.

### 🧪 Tests
- Rewrote test suite around the new oplog model (31 tests, all passing).
- Covers op generation, op application/planning, idempotency, and edge
  cases (duplicate deletes, unknown op types, TTL pruning).

### 🛠 CI
- Minor workflow fixes in `build.yml` and `release.yml`.
