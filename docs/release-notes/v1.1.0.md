# v1.1.0

## What's new

### 🎨 New extension icon
- Replaced the old red-on-green square with a Material-style design:
  blue circular background with a white bookmark glyph.

### 🐛 Popup layout fixes
- Folder icons no longer shrink when text is long (`flex-shrink: 0`).
- Action buttons ("Sync now", "Open in Chrome") no longer wrap to
  multiple lines (`white-space: nowrap`).
- General CSS cleanup for consistent spacing.

### 🔧 Sync reliability fix
- `listBookmarks` / `listFolders` now include the HTTP status code in
  error messages, allowing `withAuth` to detect expired tokens (401/403)
  and auto-refresh. Previously the generic "Failed to list bookmarks"
  message bypassed the retry logic, breaking sync after ~1 hour.

### ⚡ Rate-limit protection (429 / quota exceeded)
- Poll interval increased from 30s to 2 minutes — well within Spark
  free-tier quotas for a 2-user extension.
- Exponential backoff on 429 errors: after each consecutive failure the
  SW skips more alarm cycles before retrying (up to ~10 min gap).
- 429 is no longer misclassified as an auth error (avoids wasteful
  token refreshes that compound the quota problem).
