# v1.2.0

## What's new

### 🚫 No more background polling — zero idle Firestore reads
- Completely removed `chrome.alarms` periodic polling that was burning
  through the Firestore Spark free-tier daily quota.
- Sync is now **fully event-driven**:
  - **Outgoing**: `chrome.bookmarks.*` listeners push changes instantly.
  - **Incoming**: reconcile runs on popup open and manual "Sync now".
- This means the extension uses **zero** Firestore reads while idle.

### 🔍 Better error handling
- Quota exceeded (429 / `RESOURCE_EXHAUSTED`) now shows a clear,
  user-facing message explaining the daily limit and reset time.
- Auth errors prompt the user to sign out and back in.
- Sync errors are persisted in `chrome.storage.local` and displayed
  as a toast when the popup opens.
- Toast duration auto-extends for longer error messages.
