# Shared Bookmarks — v1.0.0

The first stable release. A Chrome MV3 extension that lets two
Google-authenticated users share a single bookmark folder in real time,
fully integrated with Chrome's native bookmark manager.

## Highlights

- **Native Chrome integration.** The shared folder lives under
  *Other bookmarks → Shared Bookmark Folder*. Anything you add, remove,
  rename, drag-and-drop or copy-paste there is synchronized.
- **Nested subfolders.** Share whole folder trees, not just flat lists.
- **Real-time-ish sync.** Background polling every ~30 s via
  `chrome.alarms` (MV3-safe), plus a manual **Sync now** button in the
  popup for instant updates.
- **One-shot 6-character invite codes.** Rooms are capped at 2 members.
- **Material 3 popup.** Membership and invite UI only — bookmarks
  themselves stay in Chrome's native manager.
- **Pinned Extension ID.** OAuth credentials remain valid even when the
  extension is reloaded from a different folder, thanks to the `key`
  field baked into `manifest.json`.
- **Transparent token refresh.** Firebase ID tokens are refreshed
  automatically on 401, so sync keeps working past the 1-hour token TTL.

## Architecture

- Plain ES2022 modules, **no bundler, no Firebase SDK, no `npm install`**
  required to run.
- Firebase REST + `chrome.identity` for auth.
- Sync engine split into:
  - `background/sync-core.js` — pure module: paths, identity,
    deduplication, diff, plan. Encodes the full
    *(local × remote × lastSynced)* decision matrix and a
    `seed-from-remote` merge mode for first-time room joins.
  - `background/service-worker.js` — MV3 I/O shell with persisted
    reconcile lock, dirty flag, echo suppression and incremental
    `lastSynced*` persistence to survive SW kills.
- Folder titles containing `/` are percent-encoded segment-by-segment
  so they don't collide with the path separator.

## Quality

- 40 unit tests on the pure sync core (`node --test`, zero
  dependencies). CI runs them on every push and tag.
- CI also greps the service worker for required listeners and
  pre-empts the "header-only file" regression that bit us once.

## Install (developer mode)

1. Download `shared-bookmarks-1.0.0.zip` from this release.
2. Unzip it to a folder.
3. `chrome://extensions` → **Developer mode** ON → **Load unpacked**.
4. Pick the unzipped folder.
5. Click the extension icon → **Continue with Google**.
6. Open the share panel and either generate a code (you create the
   room) or paste your partner's code.

The shared folder appears at *Other bookmarks → Shared Bookmark Folder*
in both browsers within ~30 seconds.
